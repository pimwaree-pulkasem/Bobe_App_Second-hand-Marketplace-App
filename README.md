# Bobe_App_Second-hand-Marketplace-App
# Silpakorn Student Marketplace

แพลตฟอร์มนี้เป็นแนวคิดแอปพลิเคชันสำหรับการซื้อ–ขายสินค้าระหว่างนักศึกษามหาวิทยาลัยศิลปากร  
วิทยาเขตสนามจันทร์เท่านั้น โดยออกแบบจาก pain point ที่พบจริงในการซื้อ–ขายสินค้ามือสองของนักศึกษา  
เช่น ความไม่น่าเชื่อถือของผู้ขาย การค้นหาสินค้ายาก และการสื่อสารที่ซับซ้อน

---

## กลุ่มผู้ใช้งาน
- นักศึกษามหาวิทยาลัยศิลปากร วิทยาเขตสนามจันทร์
- ใช้งานด้วยบัญชีที่ยืนยันตัวตนจากสถาบันเท่านั้น

---

## Pain Points และแนวทางการแก้ไข

### 1. นักศึกษาต้องค้นหาสินค้าจากหลายแพลตฟอร์ม
เช่น Facebook Group, Line, Instagram ทำให้เสียเวลาและข้อมูลกระจัดกระจาย

**ฟังก์ชันที่ช่วยแก้ไข**
- ระบบ Search & Filter  
  ช่วยให้ค้นหาสินค้าด้วย keyword, หมวดหมู่, ช่วงราคา และสภาพสินค้าได้ในแพลตฟอร์มเดียว
- Image-based Search  
  ผู้ใช้สามารถค้นหาสินค้าจากรูปภาพ เพื่อหาสินค้าที่ลักษณะใกล้เคียงกัน

**ผลลัพธ์**
- ลดการสลับไปมาระหว่างหลายแพลตฟอร์ม
- รวมตลาดซื้อ–ขายของนักศึกษาไว้ในที่เดียว

---

### 2. หาสินค้าที่ตรงความต้องการได้ยาก
ข้อมูลสินค้าไม่เป็นมาตรฐาน และต้องไล่อ่านโพสต์จำนวนมาก

**ฟังก์ชันที่ช่วยแก้ไข**
- Template การโพสต์สินค้า  
  บังคับให้ผู้ขายกรอกข้อมูลสำคัญ เช่น ชื่อสินค้า ราคา สภาพสินค้า และจุดรับสินค้า
- Fit & Style Suggestion  
  ระบบแนะนำสินค้าโดยอ้างอิงจากพฤติกรรมหรือความสนใจของผู้ใช้

**ผลลัพธ์**
- ข้อมูลสินค้าอ่านง่าย เปรียบเทียบได้ชัดเจน
- ผู้ใช้ตัดสินใจเลือกสินค้าได้เร็วขึ้น

---

### 3. ไม่มั่นใจว่าผู้ขายมีความน่าเชื่อถือหรือไม่
เสี่ยงต่อการถูกโกงหรือบัญชีปลอม

**ฟังก์ชันที่ช่วยแก้ไข**
- Verification System  
  ยืนยันตัวตนผู้ขายผ่านอีเมลมหาวิทยาลัยหรือบัตรนักศึกษา
- Blacklist และ Fraud Verification  
  ตรวจสอบและป้องกันบัญชีที่มีประวัติการโกง
- Badge แสดงความน่าเชื่อถือ  
  เช่น ผ่านการยืนยันตัวตน รีวิวดี ส่งตรงเวลา
- Rating & Review  
  ผู้ซื้อสามารถให้คะแนนและรีวิวจากประสบการณ์จริง

**ผลลัพธ์**
- ผู้ซื้อสามารถประเมินความน่าเชื่อถือก่อนตัดสินใจ
- ลดความเสี่ยงในการซื้อขายกับผู้ขายที่ไม่น่าไว้วางใจ

---

### 4. ต้องสื่อสารกับผู้ขายมากเกินไป
มีคำถามซ้ำ ๆ และต้องเปิดเผยช่องทางส่วนตัว

**ฟังก์ชันที่ช่วยแก้ไข**
- ระบบแชตภายในแอป
- Quick Questions  
  เช่น “ยังอยู่ไหม”, “รับของที่ไหน”, “ส่งได้เมื่อไหร่”

**ผลลัพธ์**
- ลดการพิมพ์ซ้ำและการสื่อสารที่ไม่จำเป็น
- ไม่ต้องเปิดเผยเบอร์โทรหรือโซเชียลมีเดียส่วนตัว

---

### 5. ได้รับสินค้าที่ไม่ตรงกับคำอธิบาย
เกิดข้อพิพาทหลังการซื้อขาย

**ฟังก์ชันที่ช่วยแก้ไข**
- Escrow System  
  เงินจะถูกพักไว้จนกว่าผู้ซื้อยืนยันว่าได้รับสินค้าตรงตามที่ตกลง
- ระบบยืนยันสินค้าก่อนโอน  
  ผู้ขายต้องส่งรูปหรือวิดีโอสินค้าพร้อม watermark และเวลาปัจจุบัน
- ระบบตรวจสอบความครบถ้วนของโพสต์สินค้า

**ผลลัพธ์**
- ลดปัญหาสินค้าไม่ตรงปก
- เพิ่มความมั่นใจให้ทั้งผู้ซื้อและผู้ขาย

---

## สรุปแนวคิดของระบบ
แพลตฟอร์มนี้มุ่งเน้นการแก้ปัญหาการซื้อ–ขายสินค้าของนักศึกษาในพื้นที่เดียวกัน  
โดยใช้การยืนยันตัวตน ความโปร่งใส และการออกแบบฟังก์ชันที่ลดความซับซ้อนของกระบวนการซื้อขาย  
เพื่อให้การแลกเปลี่ยนสินค้าระหว่างนักศึกษามหาวิทยาลัยศิลปากร วิทยาเขตสนามจันทร์  
เป็นไปอย่างปลอดภัย เชื่อถือได้ และสะดวกมากขึ้น
